* コントローラーに障害が発生していることを確認します。これは次の2つの方法で確認できます：
+
** SANtricity System Manager の Recovery Guru により、コントローラの交換が指示されました。
** コントローラーのオレンジ色の注意LEDが点灯しており、コントローラーに障害が発生していることを示しています。




* ストレージシステムの電源がオンになっていて、ホストI/O処理を実行している間に障害のあるコントローラを交換できるように、ストレージシステムの以下の条件が満たされていることを確認します：
+
** シャーシ内の2番目のコントローラのステータスは「Optimal」です。
** SANtricity System Manager の Recovery Guru の詳細エリアにある *OK to remove* フィールドに *Yes* と表示され、障害が発生したコントローラを取り外しても安全であることを示します。


* 次のものがあることを確認します。
+
** 交換するコントローラと同じ部品番号の交換用コントローラ。
** ESD リストバンドを装着するか、静電気防止処置を施しておきます。
** 静電気防止処置を施した平らな作業場所。
** コントローラから取り外す各ケーブルを識別するためのラベル。
** コントローラのSANtricity System Managerにアクセスできるブラウザを備えた管理ステーション。
+
ブラウザでコントローラのドメイン名またはIPアドレスを指定することで、SANtricity System Managerインターフェイスを開くことができます。

+
必要に応じて、コマンドラインインターフェース（CLI）を使用して一部の手順を実行することもできます。CLIパッケージ（zipファイル）をダウンロードするには、SANtricity System Manager：*Settings* > *System* > *Add-ons* を選択し、*Command Line Interface* を選択します。その後、DOS C：プロンプトなどのオペレーティングシステムのプロンプトからCLIコマンドを発行できます。
