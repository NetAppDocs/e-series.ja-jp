.手順
. この手順のダウンタイムメンテナンス期間をスケジュールします。この手順が正常に完了するまで、ストレージシステム上のデータにアクセスできません。両方のコントローラは電源投入時に同じI/Oモジュール構成である必要があるため、I/Oモジュールをアップグレードする際は電源をオフにする必要があります。
. 次のものがあることを確認します。
+
** アップグレード先の新しい I/O モジュール。
+
アップグレードするI/Oモジュールがストレージシステムでサポートされていること、各コントローラに同じタイプと数のI/Oモジュールがあること、スロットの優先順位を理解していること（新しいI/Oモジュールは各コントローラの同じスロットに取り付ける必要があります）を確認してください。

+

CAUTION: *データアクセスが失われる可能性あり* -- 互換性のない、または一致しないI/Oモジュールが存在すると、電源投入時にコントローラがロックダウンされます。

+
必要に応じて、ストレージシステムでサポートされているI/Oモジュールとスロットの優先順位を次の場所で確認できます https://hwu.netapp.com["NetApp Hardware Universe の略"^]。

** 新しいI/Oモジュールをネットワークに接続するためのケーブル。
** ESDリストバンドを着用しているか、その他の静電気対策を講じている。
** 平らで静電気のない作業面。
** コントローラのSANtricity System Managerにアクセスできるブラウザを備えた管理ステーション。
+
System Managerインターフェイスを開くには、ブラウザでターゲットコントローラのドメイン名またはIPアドレスにアクセスしてください。



. ストレージシステムの構成データベースをバックアップするには、SANtricity System Manager：
+
コントローラを削除した際に問題が発生した場合、保存したファイルを使用して設定を復元できます。システムは、障害が発生したコントローラ上のボリュームグループとディスクプールに関するすべてのデータを含む、RAID構成データベースの現在の状態を保存します。

+
.. [ サポート * （ Support * ） ]>[ * サポートセンター * （ * Support Center * ） ]>[ * 診断 * （ Diagnostics * ） ] を選択
.. [ 構成データの収集 *] を選択します。
.. *Collect* を選択します。
+
ブラウザの Downloads フォルダに、「 * configurationdata-<ArrayName>-<dateTime >.7z * 」という名前でファイルが保存されます。
