.手順
. ストレージシステムと接続されているすべてのホスト間で、I/O操作が行われていないことを確認してください。例えば、以下の手順を実行できます：
+
** ストレージからホストにマッピングされた LUN に関連するすべてのプロセスを停止します。
** ストレージからホストにマッピングされた LUN にアプリケーションがデータを書き込んでいないことを確認します。
** ストレージシステム上のLUNに関連付けられているすべてのファイルシステムをアンマウントします。
+

NOTE: ホストのI/O操作を停止するための具体的な手順は、ホストのオペレーティングシステムと構成によって異なるため、本手順の範囲外となります。環境内でホストI/O操作を停止する方法がわからない場合は、ホストをシャットダウンするか、 https://mysupport.netapp.com/site/global/dashboard["ネットアップサポート"^]担当者に問い合わせてください。

+

CAUTION: * データ損失の可能性 * - I/O 処理の実行中にこの手順を続行すると、データが失われる可能性があります。



. 各コントローラで、NV Caching Active（緑色）LEDが消灯していることを確認してください。
+
NV Caching Active（緑色）LEDが消灯している場合、キャッシュ メモリ内のデータはすべてドライブに書き込まれており、この手順を続行しても安全です。

+

NOTE: NV Caching Active（緑色）LEDが点灯している場合、キャッシュされたデータがドライブに書き込まれています。処理が完了し、NV Caching Active（緑色）LEDが消灯するまでお待ちください。ただし、LEDが5分以上点灯し続ける場合は、この手順を続行する前に https://mysupport.netapp.com/site/global/dashboard["ネットアップサポート"]お問い合わせください。

+
NV Caching Active（緑色）LEDは、コントローラー上のNVアイコンの隣にあります。

. コントローラでの処理が完了したことを確認するには、SANtricity System Managerを使用します：
+
.. *ストレージ* ＞ *進行中の操作* を選択して、概要画面を表示します。
.. すべての操作が完了し、概要画面に*現在進行中の操作はありません*と表示されるまで待ってから、次のステップに進んでください。


. ストレージシステムから定義済みのホストをすべて削除します。
+
手順については、 https://docs.netapp.com/us-en/e-series-santricity/sm-storage/delete-host-or-host-cluster.html["SANtricity System Managerでホストまたはホストクラスタを削除する"^]の手順を参照してください。
