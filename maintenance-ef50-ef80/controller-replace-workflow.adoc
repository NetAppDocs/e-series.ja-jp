= コントローラの交換ワークフロー - EF50およびEF80
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
EF50またはEF80ストレージシステムで障害が発生したコントローラを交換するには、ストレージシステムの要件を満たし、システムを準備し、障害が発生したコントローラをオフラインにし、コントローラを交換し、新しいコントローラをオンラインにして、交換プロセスを完了します。

.image:https://raw.githubusercontent.com/NetAppDocs/common/main/media/number-1.png["1つ"]link:controller-replace-requirements.html["コントローラの交換要件を確認"]
[role="quick-margin-para"]
ストレージシステムのコントローラに障害が発生していることを確認し、ストレージシステムの状態を確認し、手順に必要なツールを入手し、SANtricity System Managerにアクセスするための管理ステーションとブラウザを用意します。

.image:https://raw.githubusercontent.com/NetAppDocs/common/main/media/number-2.png["2 つ"]link:controller-replace-prepare.html["コントローラの交換の準備"]
[role="quick-margin-para"]
交換用コントローラを開梱し、 MAC アドレスをメモし、部品番号が正しいことを確認し、ストレージシステムの構成データベースをバックアップします。

.image:https://raw.githubusercontent.com/NetAppDocs/common/main/media/number-3.png["3つ"]link:controller-replace-offline-controller.html["障害のあるコントローラをオフラインにします"]
[role="quick-margin-para"]
障害のあるコントローラを交換する際は、データ損失を防ぎ、システムの安定性を確保するために、障害のあるコントローラをオフラインにしてください。

.image:https://raw.githubusercontent.com/NetAppDocs/common/main/media/number-4.png["4."]link:controller-replace-move-hardware.html["障害のあるコントローラを交換します"]
[role="quick-margin-para"]
キャッシュメモリ内のデータがドライブに書き込まれていることを確認し、障害コントローラからすべてのケーブルを抜き、コントローラを取り外し、コンポーネントを交換用コントローラに移動し、交換用コントローラを取り付け、すべてのケーブルを再接続します。

.image:https://raw.githubusercontent.com/NetAppDocs/common/main/media/number-5.png["5 つ"]link:controller-replace-online-controller.html["交換用コントローラをオンラインにします"]
[role="quick-margin-para"]
交換用コントローラのIPアドレスを確認し、コントローラをオンラインにして、ストレージシステムのステータスがOptimalであることを確認してください。

.image:https://raw.githubusercontent.com/NetAppDocs/common/main/media/number-6.png["6"]link:controller-replace-complete-replacement.html["コントローラの交換を完了します"]
[role="quick-margin-para"]
最新バージョンの SANtricity OS がインストールされていることを確認し、ボリュームが所有コントローラに戻されていることを確認し、システムサポートデータを収集して、障害が発生した部品をNetAppに返却してください。
